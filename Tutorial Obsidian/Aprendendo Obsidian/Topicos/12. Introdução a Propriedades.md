## **Introdução a propriedades**

A partir da versão 1.4 do Obsidian a ferramenta aprimorou a capacidade do aplicativo de gerenciar metadados dentro das notas.

Adicionar metadados a nota permite uma organização e categorização, não só em notas triviais, mas também para utilização em templates e plugins.

Podemos pensar em metadados como a capa de um livro contendo informações detalhadas sobre o conteúdo apresentado no seu interior. Uma vez que você utilize de forma correta, você pode potencializar seu segundo cérebro com informações para que o seu vault se transforme em um poderoso banco de dados de ideias.

Antes da versão 1.4 da ferramenta, inseríamos essas informações digitando três traços na primeira linha da nota.

Um exemplo de metadados em uma nota de resumo de livro:

![](https://api-club-file.cb.hotmart.com/public/v5/files/573f2908-c5ca-4a62-b8bc-389ab8fac89c)​  

Agora, após versão 1.4, ao digitarmos os três traços na nota, o editor de propriedades é exibido:

![](https://api-club-file.cb.hotmart.com/public/v5/files/20d042e3-a0f0-40b7-bf2b-4f775cdf0419)

  

Existem alguns atalhos para a inserção das propriedades:

Na própria nota através do Botão **Add property**

**![](https://api-club-file.cb.hotmart.com/public/v5/files/f978e8c5-15b8-4453-b997-74bee6648c8b)**

Na paleta de comandos com **CTRL/CMD + P** através da opção **Add file property**​

**![](https://api-club-file.cb.hotmart.com/public/v5/files/70866a6d-4d69-4d88-bf3b-4ec1f0b3472c)**

Através do atalho **Ctrl + ;**

Ao clicar com o botão direito em cima de uma propriedade você pode escolher o tipo de dado que será inserido na propriedade.

![](https://api-club-file.cb.hotmart.com/public/v5/files/ddeb2020-7265-4167-a8ad-5b65c39a22f4)

  

Dessa forma, inserir metadados se tornou uma operação mais visual e prática, um tanto no-code para quem não tinha o hábito de inserir na forma textual.

A inserção de propriedades torna possível inserir urls e links para notas internas no obsidian.Isso torna possível centralizar nos metadados as informações de referências de uma nota, sejam sites ou outras notas internas, geralmente aquelas informações que você coloca no final de cada nota como complemento.

## **Forma de visualizar**

É possível em uma nota ver as  propriedades em sua forma textual. Basta para isso clicar no canto superior direito e no menu selecionar Source mode.

![](https://api-club-file.cb.hotmart.com/public/v5/files/1ed1f4e6-f6bd-49b3-a820-709fc558f994)

  

Existe uma configuração caso você deseje alterar a forma como as propriedades são vistas em todo o cofre. Ela está no nas configurações em **Editor / Properties in document**.

![](https://api-club-file.cb.hotmart.com/public/v5/files/067a8699-528c-46f3-a74d-e6ad8bcf25e6)

  

Os valores possíveis de Properties in document são:

**Visible**: As propriedades são visíveis no inicio da nota.

**Hidden**: As propriedades não ficam visíveis na nota.

**Source**: As propriedades ficam visíveis na forma textual.

## **Visualização das propriedades em nota individual e em todo o cofre**

Na paleta de comando (**Ctrl/Cmd + P**), existem duas novas opções para habilitar painéis referentes as propriedades.

Para que essas opções fiquem ativas é necessário habilitar nas configurações o Plugin Nativo chamado **Properties view**, disponível em **_Core Plugins / Properties view_.**

**![](https://api-club-file.cb.hotmart.com/public/v5/files/ca711eac-d7fe-465d-8f1a-5d17b990ac3b)**

As opções na paleta de comando são:

**Show all properties**: Mostra todas as propriedades em uso no cofre.

**Show file properties**: Mostra as propriedades da nota selecionada.

![](https://api-club-file.cb.hotmart.com/public/v5/files/ffa93fbf-faa3-45fa-9447-4c14e92d1fcf)

  

Essa opção no sidebar direito é muito útil para quando se configura o cofre para não mostrar os metadados na parte inicial da nota. A edição então pode ser feita nessa nova janela.

![](https://api-club-file.cb.hotmart.com/public/v5/files/4299ab45-a9dc-4f3e-835e-e47eddb4ee6b)

  

A opção **Show all properties** mostra todas as propriedades existentes no cofre e quantas vezes ela é utilizada.

![](https://api-club-file.cb.hotmart.com/public/v5/files/9c7606ca-19c0-4732-ae0d-747f13ba7428)

  

Ao clicar em alguma das propriedades relacionadas no sidebar, o Obsidian exibe as notas que estão utilizando aquela propriedade. O painel de pesquisa, localizado no sidebar esquerdo, exibe o filtro que está sendo feito e todas as notas pertencentes.

Isso revela que com as propriedades, o Obsidian também ganhou a capacidade de mais um filtro, que é utilizado no formato
```
[nomePropriedade:valor]
```

**![](https://api-club-file.cb.hotmart.com/public/v5/files/c4479942-7fb2-430d-abd1-250b9242b3b3)**

Dessa forma você pode localizar rapidamente no seu segundo cérebro notas que estão dentro de uma determinada categoria de pesquisa.

**Importante: Não se deve usar nenhum tipo de formatação markdown nas propriedades e nem nos valores relacionados.**

Propriedades Nativas do Obsidian: **aliases, cssclasses e tags**. Vamos entender para que cada uma delas funciona.

## **Propriedade Aliases**

Propriedade do tipo lista, onde você pode colocar vários valores para apelidos curtos a sua nota. Em todo o cofre, quando quiser adicionar um link interno, irão aparecer o nome das notas e os apelidos.

Exemplo de apelidos para a nota criada de resumo de livro:

![](https://api-club-file.cb.hotmart.com/public/v5/files/40bddb0a-faa1-4f78-a529-21dcdf6f30a8)

  

Ao digitar o comando de link em outra nota [[ ]], aparece a nota e os apelidos como possibilidade de link**.**

**![](https://api-club-file.cb.hotmart.com/public/v5/files/8d547fd0-40e2-40f7-9f56-1fac97cf4c32)**

## **A propriedade cssclasses**

Essa propriedade permite que você adicione classes css para customizar a nota individualmente.

Os arquivos css que possuem as classes precisam estar em uma pasta padrão dentro do cofre, esses arquivos no Obsidian são chamados de **snippets**.

Para achar a pasta e poder criar arquivos css é só acessar o menu **Appearance / CSS Snippets.**

**![](https://api-club-file.cb.hotmart.com/public/v5/files/01fa9318-21d7-43fd-baa1-d57dbb6643aa)**

Ao clicar no ícone de pasta o Obsidian irá abrir o gerenciador de arquivos já no local onde você pode criar ou colocar seus arquivos css. Após isso é só clicar no botão de atualizar e clicar para ativar cada um dos nomes que aparecerem na lista.

![](https://api-club-file.cb.hotmart.com/public/v5/files/2e79d185-3439-4eb3-899f-b94021750626)

  

## **Propriedade Tags**

A propriedade Tags é do tipo lista e pode conter vários valores. Nesse caso você pode inserir os nomes das tags diretamente, sem o #, pois estando nessa propriedade o Obsidian já vai tratar e exibir em todas as visualizações.

Exemplo de tags criadas para o arquivo de resumo de livro.

![](https://api-club-file.cb.hotmart.com/public/v5/files/1355224a-0a18-4811-8b93-478d9060b77b)

  

No sidebar direito, clicando no ícone de tags, já aparecem os valores que foram inseridos na nota.

  

Dessa forma o Obsidian potencializa a operação com metadados e agora de forma mais visual.

Agora que você aprendeu como utilizar esse recurso de forma produtiva e eficaz, é hora de colocar em prática para que seu segundo cérebro se torne seu aliado, muito mais organizado e poderoso.
